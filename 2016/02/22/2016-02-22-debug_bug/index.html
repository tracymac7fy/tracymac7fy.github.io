andorid studio debug模式的一个坑

2016-02-22

最近刷了机，用studio运行程序并且开启debug模式时，发现了一个巨坑！
在开启debug模式的时候，总是提示我adb无法开启，手动杀死后重启。
用cmd命令行
>
netstat -ano|findstr “5037” //看看哪个进程占用了5037端口
tasklist|findstr “xxx” //通过进程号查看该进程
taskkill /f /t /im adb.exe // 杀死该进程

这样·排查后发现是qq浏览器的锅，占有了该端口
怒卸载！！！
重新开启debug模式，这时又出现了新的问题：debug info can be unavailable。。。。
网上找了好久，还在google issue上找到同样的问题，正当以为是studio的bug，束手无策时
想起了开发者模式！
在开发者模式–》选择调试应用–》微云
然后再开启debug模式，成功！！

搞了这么久，居然是这么简单解决 。。。囧。。。。
